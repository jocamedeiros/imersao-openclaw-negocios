Bloco 4 — Skill Creator

O agente que cria outros agentes

Descreva um processo → skill funcional com QA automático → pronta pra usar

Prompt
Temporário
  • ✗ Morre quando a sessão fecha
  • ✗ Explica tudo de novo toda vez
  • ✗ Vive na cabeça de alguém
  • ✗ Impossível delegar ou agendar
Skill
Permanente
  • ✓ Salva no Cérebro (Git)
  • ✓ Qualquer agente acessa
  • ✓ Roda quando quiser / agendar
  • ✓ Melhora a cada uso
📸
Modo 1
Captura de sessão
Acabou de fazer algo com o agente? Ele lê o histórico e monta a skill sozinho.
Exemplo
"Transforma isso que a gente fez em skill"
📋
Modo 2
Colar processo
Tem um passo-a-passo anotado? Cola no chat — o agente identifica as etapas.
Exemplo
"Todo mês eu acesso o painel, exporto o relatório e mando por email"
💡
Modo 3
Descrever ideia
Ideia vaga? O agente faz as perguntas certas até montar a skill completa.
Exemplo
"Quero algo que avise quando um lead esfria"
O que acontece quando você pede /criar-skill
🎯
Detecta modo — captura, workflow ou entrevista
↓
📝
Gera SKILL.md — workflow, edge cases, exemplos
↓
🔍
QA automático — 10 checks antes de salvar
↓
💾
Deploy — salva no Cérebro + evals de teste
↓
⚡
Pronta — qualquer agente usa, agenda, evolui
QA automático — 10 checks
✓ Nome em kebab-case
✓ Triggers suficientes
✓ Passos claros e imperativos
✓ Exemplos reais (in/out)
✓ Edge cases cobertos
✓ Output format definido
✓ Sem linguagem vaga
✓ Sem credenciais expostas
✓ Boundaries negativos
✓ Evals preparados
O que é gerado
skill-name/
├── SKILL.md ← workflow, regras, edge cases
├── evals/
│ └── evals.json ← testes automáticos
├── references/ ← docs de apoio (opcional)
└── scripts/ ← código executável (opcional)
💡 Conhecimento sai da cabeça → vira receita → melhora a cada uso | Skills não saem perfeitas na 1ª vez — e tá tudo bem. Elas evoluem com o uso real.
